## 2) Backend – Request Validation Layer

To keep the controller clean and avoid repeating validation rules, three dedicated request classes were created.

### 2.1 `StoreDossierRequest`

Created to validate dossier creation input:

* `assured_identifier` → required, string, exactly 8 characters
* `episode_description` → nullable, string, max 255
* `notes` → nullable, string, max 1000

**Purpose:** Make sure every newly created dossier starts with valid business data.

---

### 2.2 `UpdateDossierRequest`

Created to validate dossier metadata updates:

* `episode_description` → sometimes, nullable, string, max 255
* `notes` → sometimes, nullable, string, max 1000

**Purpose:** Allow partial updates safely. Using `array_key_exists` in the controller makes sure missing fields do not overwrite existing values with `null`.

---

### 2.3 `AttachDocumentsRequest`

Created to validate dossier attachment requests:

* `document_ids` → required, non-empty array
* Each item must be:
  * required
  * integer
  * distinct
  * existing in the `documents` table

**Purpose:** Prevent malformed attach requests before the business logic runs.

## 3) Backend – Dossier API Controller

Created `app/Http/Controllers/Api/DossierController.php` with the following methods:

* `index()`
* `store(StoreDossierRequest $request)`
* `show(Dossier $dossier)`
* `update(UpdateDossierRequest $request, Dossier $dossier)`
* `destroy(Dossier $dossier)`
* `attachDocuments(AttachDocumentsRequest $request, Dossier $dossier)`

The controller follows the same project direction already used in the API:

* Authenticated access with `auth:sanctum`
* Explicit ownership checks with `abort(403)`
* Explicit JSON responses
* Business-rule enforcement in the controller and model
* Transaction protection for multi-record operations

## 4) API Routes Added

Added a new authenticated dossier route group in `routes/api.php`:

```php
Route::middleware('auth:sanctum')->prefix('dossiers')->group(function () {
    Route::get('/', [DossierController::class, 'index']);
    Route::post('/', [DossierController::class, 'store']);
    Route::get('/{dossier}', [DossierController::class, 'show']);
    Route::put('/{dossier}', [DossierController::class, 'update']);
    Route::delete('/{dossier}', [DossierController::class, 'destroy']);
    Route::post('/{dossier}/attach-documents', [DossierController::class, 'attachDocuments']);
});
```

**Result:** The dossier domain is now reachable from the frontend or Postman through a protected REST API.

## 5) Endpoint Behavior Implemented

### 5.1 `index()` – List user dossiers

* Filters by `created_by = auth()->id()`
* Orders by latest
* Includes `documents_count`

### 5.2 `store()` – Create dossier

* Uses validated input from `StoreDossierRequest`
* Forces `created_by = auth()->id()`
* Initializes `status = DossierStatus::RECU`

### 5.3 `show()` – View one dossier

* Enforces ownership check
* Eager loads attached documents
* Exposes `display_total` using the model method `getDisplayTotal()`

### 5.4 `update()` – Update dossier metadata

* Blocks updates if the dossier is frozen (`isTotalFrozen()`)
* Safely updates only `episode_description` and `notes`
* Status transitions are intentionally left for future workflow actions

### 5.5 `destroy()` – Delete dossier

* Blocks deletion if `canBeDeleted()` returns false

## 6) The Critical Endpoint: `attachDocuments()`

This endpoint is the most sensitive part of Day 17 because it connects documents, dossiers, and reimbursement totals.

The flow works like this:

1. Validate the request payload using `AttachDocumentsRequest`.
2. Open a `DB::transaction()`.
3. Lock the dossier row with `lockForUpdate()`.
4. Verify dossier ownership and frozen state.
5. Loop through the requested document IDs:

   * Lock each document row with `lockForUpdate()`
   * Check that the document belongs to the authenticated user
   * Check that the document status is exactly `VALIDATED`
   * Check that the document is not already attached to another dossier
6. Attach the valid documents by updating `dossier_id`.
7. Recalculate the total with `$dossier->updateTotal()`.
8. Return a success response with the number of attached documents and the updated total.

**Result:** Document attachment is protected against race conditions and stays consistent with the reimbursement business rules.
