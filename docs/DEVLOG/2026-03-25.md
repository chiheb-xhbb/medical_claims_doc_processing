## 7) Backend Stabilization â€“ Attachment Persistence Fix

### 7.1 Bug observed

During end-to-end testing, the attach action initially showed a success message in the UI, but:

* the selected document did not appear in the attached documents section
* `montant_total` stayed `0.000`
* refreshing the page confirmed that the document had not actually been linked to the dossier

### 7.2 Root cause

The dossier attachment logic in `DossierController::attachDocuments()` attempted to assign the foreign key through mass assignment.

Because `dossier_id` was intentionally not exposed for general mass assignment, the field was not persisted even though the request completed successfully.

### 7.3 Fix applied

The controller was corrected to use explicit assignment for the dossier link before saving the document.

This preserved the original security intention while making the attach operation actually persist.

### 7.4 Result

After the fix:

* selected validated documents are truly attached
* the dossier detail endpoint returns the updated attached documents
* `updateTotal()` runs against the correct attached set
* the total displayed in the UI refreshes correctly

This was the critical stabilization point that closed the Day 18 workflow.
