# 3) Backend – Idempotent Async Job Processing

## 3.1 Preventing Status Regression

Added early guard inside `ProcessDocumentJob::handle()`:

```php
if ($document->status === DocumentStatus::VALIDATED) {
    return;
}
```

### Purpose

Once a document is VALIDATED, it becomes immutable.
No retry or async execution may downgrade its state.

---

## 3.2 Preventing Duplicate Extraction Creation

```php
$existingExtraction = Extraction::where('document_id', $this->documentId)
    ->where('version', self::EXTRACTION_VERSION)
    ->exists();

if ($existingExtraction) {
    return;
}
```

### Purpose

* Prevent duplicate extraction version 1
* Safe retry handling
* Job execution becomes idempotent

---

## 3.3 Unique Job Enforcement

Already implemented:

```php
implements ShouldBeUnique
```

```php
public function uniqueId(): string
{
    return (string) $this->documentId;
}
```

### Result

* Only one job per document ID allowed concurrently
* Eliminates race conditions
* Guarantees concurrency-safe processing

---

## 3.4 Protecting Against Incorrect Failure State

Inside `failed()`:

```php
if ($document->status === DocumentStatus::VALIDATED) {
    return;
}
```

### Purpose

If validation occurred during retry attempts, the job must not overwrite the final VALIDATED state with FAILED.

# 4) Frontend – Polling Stability Improvement

## 4.1 Issue Identified

Original polling logic used `currentPage` directly inside `setInterval()`.

Due to React closure behavior, this could lead to:

* Stale state usage
* Incorrect page polling
* Potential infinite refresh loops

---

## 4.2 Implemented Solution

Introduced `useRef` stabilization:

```javascript
const currentPageRef = useRef(currentPage);

useEffect(() => {
  currentPageRef.current = currentPage;
}, [currentPage]);
```

Polling call updated:

```javascript
const newDocs = await fetchDocuments(currentPageRef.current);
```

---

## 4.3 Result

* No stale closure issues
* Polling always targets the correct page
* Interval clears automatically when no documents require updates
* Stable and predictable UI behavior
