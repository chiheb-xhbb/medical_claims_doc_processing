## 2) API Contract Integrity – `doc_type` Persistence

### 2.1 Problem Identified

The API accepted a `doc_type` parameter but did not consistently persist it across the backend layer. This violated strict contract integrity principles: *An API must never accept parameters that are silently ignored.*

### 2.2 Fix Implemented

* **DocumentController:** Ensured `doc_type` is explicitly passed to the service layer.
* **DocumentService:** Persisted `doc_type`, defaulting to `medical_invoice`.
* **FastAPI (`main.py`):** Set a strict fallback default:

```python
doc_type: Optional[str] = Form("medical_invoice")

```

### 2.3 Result

* API contract is now strictly respected.
* Domain is explicitly scoped to invoices for the MVP.
* No silent parameter loss across microservices.

## 3) Transactional Integrity & File I/O Refactoring

### 3.1 Problem Identified

File storage previously occurred *inside* a database transaction.
**Risks:** Long-held DB connections, scalability bottlenecks, orphaned files if the transaction fails, and mixed I/O + DB responsibilities.

### 3.2 Refactor Applied

Separated the I/O operation from the Database transaction to ensure clean rollback:

```php
$path = Storage::putFileAs(...);

try {
    DB::beginTransaction();

    $document = Document::create([...]);

    ProcessDocumentJob::dispatch($document->id)->afterCommit();

    DB::commit();
} catch (\Exception $e) {
    DB::rollBack();
    Storage::delete($path); // Cleanup orphaned file
    throw $e;
}

```

### 3.3 Improvements Achieved

* File is written before the DB transaction begins.
* Transaction window is heavily minimized.
* Clean separation of concerns (I/O vs. Persistence).

## 4) AIService Resource Leak Prevention

### 4.1 Problem Identified

The `fopen()` handle used to send files to the AI microservice was not guaranteed to close if the HTTP attachment failed. This risked file descriptor exhaustion and memory leaks.

### 4.2 Fix Implemented

Wrapped the FastAPI HTTP call inside a `try/finally` block:

```php
$handle = fopen($fullPath, 'r');

try {
    $response = Http::attach('file', $handle, basename($filePath))
        ->post(...);
} finally {
    fclose($handle);
}

```

### 4.3 Result

* Guaranteed file handle closure.
* Resource lifecycle is strictly controlled.
* Architecturally sound memory safety established.

## 5) OCR Binary Path Externalization

### 5.1 Problem Identified

The Tesseract executable path was hardcoded for local Windows development (`C:\Program Files\Tesseract-OCR\tesseract.exe`), making the AI service non-portable.

### 5.2 Solution

Updated `main.py` to prioritize environment variables with an OS-level fallback:

```python
tesseract_path = os.getenv("TESSERACT_CMD")

if tesseract_path:
    pytesseract.pytesseract.tesseract_cmd = tesseract_path
elif os.name == 'nt':
    pytesseract.pytesseract.tesseract_cmd = r'C:\Program Files\Tesseract-OCR\tesseract.exe'

```

### 5.3 Result

* Environment-driven configuration.
* Container-ready architecture design.

## 6) AI Logic: Tunisian Dinar Extraction Improvement

### 6.1 Enhancement Applied

Improved the regular expressions to accurately support the Tunisian fiscal context:

* Handled `T.T.C.` formatting.
* Supported Tunisian millimes (3 decimal places).
* Handled optional `DT` and `TND` suffixes.

```python
amount_pattern = r'(?:total\s*t\.?t\.?c\.?|ttc|montant)[^\d]*(\d+[.,]\d{2,3})'

```

*Implemented a greedy match to always select the last occurring total to prevent grabbing partial sub-totals.*

### 6.2 Result

* Full compliance with Tunisian fiscal formatting.
* Extraction accuracy significantly improved.
