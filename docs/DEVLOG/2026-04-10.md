## 2) Core Architectural Decision

### 2.1 No Public File Access

Rejected approach:

```
React → direct file URL
```

Adopted approach:

```
React → Laravel → private storage
```

### Reasoning

* preserve RBAC enforcement
* avoid storage exposure
* maintain Laravel as source of truth
* align with enterprise document systems

---

### 2.2 Blob-Based Frontend Access

Because authentication uses SPA tokens (Axios):

Direct navigation like:

```js
window.open('/api/documents/12/view')
```

would fail (no auth header).

Adopted solution:

```js
api.get('/documents/12/view', { responseType: 'blob' })
```

Then:

* preview via object URL
* download via temporary anchor

### Result

* no auth redesign
* full compatibility with current frontend
* secure access preserved

## 3) Backend – Protected File Streaming

### 3.1 New Endpoints

Added in `routes/api.php`:

```php
Route::get('/{document}/view', [DocumentController::class, 'view']);
Route::get('/{document}/download', [DocumentController::class, 'download']);
```

These routes are inside:

```php
Route::middleware(['auth:sanctum', 'active.user'])
```

---

### 3.2 Authorization Logic

Reused and extended document access rules:

```php
if (! $this->canAccessDocumentFile($user, $document)) {
    return response()->json([
        'message' => 'Unauthorized access to this document.',
    ], 403);
}
```

Key point:

* **No new permission model introduced**
* fully aligned with existing RBAC + dossier visibility

---

### 3.3 File Existence Hardening

```php
if (! Storage::disk('local')->exists($document->file_path)) {
    return response()->json([
        'message' => 'Source file not found.',
    ], 404);
}
```

Prevents:

* broken preview
* internal path leakage
* runtime exceptions

---

### 3.4 Preview Logic (MIME-aware)

```php
$previewableMimes = ['application/pdf', 'image/jpeg', 'image/jpg', 'image/png'];

if (! in_array($mimeType, $previewableMimes, true)) {
    return Storage::disk('local')->download(...);
}
```

Then:

```php
return response()->streamDownload(function () use ($document) {
    echo Storage::disk('local')->get($document->file_path);
}, $document->original_filename, [
    'Content-Type' => $mimeType,
    'Content-Disposition' => 'inline; filename="..."',
]);
```

---

### 3.5 Download Logic (Original Filename)

```php
return Storage::disk('local')->download(
    $document->file_path,
    $document->original_filename ?? 'document'
);
```

### Result

* user downloads: `invoice_march_2026.pdf`
* not internal hashed filenames

## 4) Frontend – Document Access Service

Created:

`src/services/documentAccess.js`

### 4.1 Preview (Popup-safe implementation)

```js
const previewWindow = window.open('', '_blank');

const response = await api.get(`/documents/${id}/view`, {
  responseType: 'blob'
});

const url = URL.createObjectURL(response.data);
previewWindow.location.href = url;
```

Fallback:

```js
const anchor = document.createElement('a');
anchor.href = url;
anchor.target = '_blank';
anchor.click();
```

### Key improvement

* prevents browser popup blocking after async calls
* ensures reliable preview behavior

---

### 4.2 Download

```js
const response = await api.get(`/documents/${id}/download`, {
  responseType: 'blob'
});

const url = URL.createObjectURL(response.data);

const anchor = document.createElement('a');
anchor.href = url;
anchor.download = filename;
anchor.click();
```
